# 第10回 思考発話法による評価

ペアになって互いの作品を体験し、思考発話法（Think-Aloud Protocol）を使って改善点を発見します。

## 今回のゴール

- 思考発話法の手法を理解し、実践する
- 他の受講者に作品を体験してもらい、ユーザーの視点からフィードバックを得る
- 発見した課題をもとに改善計画を立てる

## 準備するもの

| アイテム | 説明 |
|---------|------|
| 自分の作品 | 音が出てセンサで操作できる状態にしておく |
| ノート・筆記用具 | 観察記録用 |
| 作品説明シート | 前回作成したもの |

---

## 思考発話法とは

### 概要

**思考発話法（Think-Aloud Protocol）** は、ユーザビリティ評価の代表的な手法の一つです。体験者に作品を使ってもらいながら、**頭の中で考えていることをそのまま声に出してもらう** ことで、直感的な反応や困惑ポイントを発見します。

### なぜ思考発話法を使うのか

| 通常のフィードバック | 思考発話法 |
|-------------------|----------|
| 体験後に感想を聞く | 体験中にリアルタイムで思考を聞く |
| 記憶に頼るため正確でないことがある | その瞬間の率直な反応が得られる |
| 「良かったです」と曖昧になりがち | 「これは何だろう？」と具体的な反応がわかる |
| 問題点を忘れてしまうことがある | 操作に迷った瞬間がわかる |

::: tip 思考発話法のメリット
- 少ない人数（1〜2人）でも有用なフィードバックが得られる
- 作品の「わかりにくさ」が具体的に発見できる
- 制作者が気づかなかった問題や面白さが見つかる
:::

---

## 実施手順

### 役割分担

- **評価者（体験する人）**: 相手の作品を体験しながら、考えていることを声に出す
- **観察者（制作者）**: 評価者の発言・行動・表情を観察し、メモを取る

### 手順

#### 1. 準備（2分）

1. 作品をセットアップする（micro:bit の接続、Pd パッチの起動）
2. キャプションや作品説明シートは **見えない場所に置く**（最初は説明なしで体験してもらう）
3. 観察記録シートを手元に用意する

::: warning 重要：最初は説明しない
思考発話法では、最初に操作方法を説明しません。「説明なしでどこまで伝わるか」を確認することが目的です。評価者が完全に行き詰まった場合のみ、最小限のヒントを出しましょう。
:::

#### 2. 思考発話法の説明（1分）

観察者は評価者に以下のように依頼します。

> 「これから私の作品を体験してもらいます。体験中は、**頭の中で考えていることをそのまま声に出してください**。例えば、『これは何だろう？』『ここを触ったら音が変わった』『次はこうしてみよう』など、思ったことをそのまま言葉にしてもらえると助かります。」

#### 3. 体験と観察（5分）

**評価者がやること:**
- 作品に自由に触れる
- 考えていることを声に出し続ける
- 「今何をしようとしている」「何が起きた」「どう感じた」を話す

**観察者がやること:**
- 評価者の **発言内容** をメモする
- 評価者の **表情** を観察する（笑顔、困惑、驚きなど）
- 評価者が **操作に迷った場面** を記録する
- 沈黙が続いたら「今何を考えていますか？」と促す
- 評価者が完全に行き詰まったら最小限のヒントを出す

::: tip 促し（プロンプティング）のフレーズ例
- 「今何を考えていますか？」
- 「何をしようとしていますか？」
- 「今の操作でどんなことが起きましたか？」
- 「何か気になったことはありますか？」
:::

#### 4. 体験後のインタビュー（3分）

体験が終わったら、以下の質問をしましょう。

1. **全体の印象** --- 「体験してみてどうでしたか？」
2. **面白かった点** --- 「特に面白いと感じた部分はどこですか？」
3. **わかりにくかった点** --- 「操作に迷った部分はありましたか？」
4. **改善のアイデア** --- 「こうなったらもっと面白いと思うことはありますか？」

#### 5. 役割交代

評価者と観察者を交代して、同じ手順を繰り返します。

## 記録のポイント

### 分類して整理する

記録した内容を以下の3つに分類しましょう。

#### 良かった点

- 「面白い！」と言った場面
- 笑顔や驚きの反応があった場面
- 直感的に操作できた場面

#### わかりにくかった点

- 「これは何？」と言った場面
- 操作方法がわからず止まった場面
- 意図と異なる操作をした場面
- 「あれ？」と困惑した場面

#### 改善のアイデア

- 評価者から出たアイデア
- 観察から気づいた改善の可能性
- 「こうすればもっと伝わるかも」というひらめき

## 改善計画の立案

### 課題の優先順位付け

発見した課題を以下の基準で優先順位をつけます。

| 優先度 | 基準 | 例 |
|-------|------|-----|
| **高** | 操作方法が全くわからない | センサの使い方が直感的でない |
| **高** | 作品のコンセプトが伝わらない | 何を表現しているかわからない |
| **中** | 操作はできるが改善の余地がある | 音の変化がもう少し大きいとよい |
| **中** | 体験の満足度を上げられる | エフェクトを追加するとよい |
| **低** | 細かい調整 | 音量バランスの微調整 |

### 改善策のリストアップ

各課題に対して、技術的に対応可能な改善策を考えましょう。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
改善計画シート
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

課題1:（例: 傾けても音の変化が小さい）
  優先度: 高
  改善策: センサ値のマッピング範囲を広げる
  Pd での対応: [* 3] でセンサ値を3倍にする

課題2:（例: 何を表現しているかわからなかった）
  優先度: 高
  改善策: 環境音（波の音）をバックグラウンドに追加する
  Pd での対応: [readsf~] で波の音のファイルをループ再生する

課題3:（例: ボタンの機能がわからない）
  優先度: 中
  改善策: ボタンを押したときにLEDで表示を変える
  micro:bit での対応: ボタンAが押されたときにアイコンを変更する

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

::: tip 次回以降に向けて
改善計画をもとに、第11回・第12回で作品をブラッシュアップしていきます。すべての課題を解決する必要はありません。**優先度「高」の課題** を中心に対応しましょう。
:::
