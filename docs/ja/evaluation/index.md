# 評価手法の概要

作品を制作したら、それが「どう伝わったか」「どう体験されたか」を客観的に評価することが大切です。このセクションでは、授業で使用する2つの主要な評価手法を紹介します。

## なぜ評価するのか？

自分の作品について「良い感じにできた」「面白いと思う」という主観的な感覚は大事ですが、それだけでは不十分です。評価を行うことで：

- **意図が伝わっているか確認できる** — 「怖い雰囲気」を目指したのに「楽しい」と感じられていたら、改善の手がかりになる
- **改善点が見つかる** — 数値やグラフで弱点が可視化される
- **他の人と共有できる** — 「なんとなく良い」ではなく、データで説明できる

## はかるもの

この授業では、主に以下の2つを測定します。

### 印象

作品から受ける印象を、複数の形容詞を使って評価します。「明るい↔暗い」「派手な↔地味な」のような対立する言葉のペアを使います。

→ **[SD法（Semantic Differential法）](./sd-method)** で測定

### UX（ユーザー体験の品質）

作品を使った体験の品質を評価します。UXは大きく2つの側面があります：

| 品質 | 意味 | 例 |
|------|------|------|
| **実用的品質**（Pragmatic Quality） | 使いやすさ・わかりやすさ | 「操作が直感的」「すぐに使い方がわかった」 |
| **感性的品質**（Hedonic Quality） | 楽しさ・魅力・新しさ | 「ワクワクする」「今までにない体験」 |

→ **[UEQ-S（User Experience Questionnaire - Short version）](./ueq-s)** で測定

## その他の評価手法（参考）

この授業では主にSD法とUEQ-Sを使いますが、研究ではこのような手法もよく使われます：

| 手法 | 測定対象 | 特徴 |
|------|----------|------|
| SAM | 感情（快不快・覚醒度） | イラストで直感的に回答できる |
| AttrakDiff | UX（実用的品質・感性的品質） | UEQ-Sの元になった手法 |
| POMS2 | 気分（ネガティブな側面） | 医療・心理学でよく使用 |
| 半構造化インタビュー | 体験の詳細 | あらかじめ質問を用意し、回答に応じて深掘り |
| テキストマイニング | 自由記述の傾向 | 語句の頻度や共起性を分析 |

## くらべる・ほりさげる

評価をより深くするための方法：

- **要素を変えて比較する** — 作品内の音や演出を変えたバージョンを用意して比較
- **媒体を変えて比較する** — ヘッドホン vs スピーカー、モニター vs プロジェクタ
- **既存の作品と比較する** — 似たコンセプトの作品やアプリとの比較
- **因子分析** — SD法の結果から、印象のカテゴリを統計的に抽出
